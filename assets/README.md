## Installation (Manual via Zoho Admin)

### Step 1: Open Zoho Commerce Admin
1. Go to https://commerce.zoho.com
2. Select **SAAutomotive** organization
3. Navigate to **Settings** → **Site Settings** → **Third Party Code**

### Step 2: Add Header Code
Copy and paste this into the **Header Code** section:

```html
<script src="https://cdn.jsdelivr.net/gh/Stan2891/bmparts-ai@main/assets/category-icons-injector.js" defer></script>
```

### Step 3: Save and Publish
1. Click **Save**
2. Go to **Site** → **Publish** to push changes live

## Technical Details

### Responsive Breakpoints
- **Desktop (>768px)**: 45px icon, 56×56 container
- **Tablet (≤768px)**: 42px icon, 52×52 container  
- **Mobile (≤480px)**: 38px icon, 48×48 container

### Features
- Lazy loading (`loading="lazy"`)
- MutationObserver for SPA navigation
- Respects `prefers-reduced-motion`
- Graceful fallback (no icon shown if no match)
- Case-insensitive category matching

### Selectors Targeted
The script looks for elements matching:
- `.category-card`
- `.category-item`
- `.category-grid-item`
- `.zs-category-item`
- `.zs-category-card`

If your theme uses different classes, update `CATEGORY_SELECTORS` in the script.

## Troubleshooting

### Icons not showing
1. Check browser console for errors
2. Verify header code is saved in Zoho
3. Verify site is published
4. Clear browser cache (Ctrl+Shift+R)
5. Check if category names match the mapping

### Wrong icon size
The CSS uses `!important` for sizing. If overridden by theme, increase specificity in the injector styles.

### CDN caching
jsDelivr caches for up to 7 days. To force refresh after updating:
```
https://cdn.jsdelivr.net/gh/Stan2891/bmparts-ai@latest/assets/category-icons-injector.js
```
(Use `@latest` instead of `@main`)
